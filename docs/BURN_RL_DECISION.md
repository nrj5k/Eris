# Burn-RL Decision

The burn-rl crate was removed from the project because it used a pre-release version (0.21.0-pre.2) that conflicted with the stable Burn 0.20 dependency, causing version resolution issues in Cargo. To resolve these conflicts, we removed burn-rl from Cargo.toml and reverted to using Burn 0.20 as the supported version. Files that imported traits from burn_rl (such as those in the training module) will need to be either rewritten to use stable Burn APIs or removed entirely if they cannot be adapted. The manual training loop implementation in train.rs continues to work correctly without burn_rl, providing a functional alternative for training operations. The recommendation is to use the manual training loop approach for now until burn-rl releases a stable version that is compatible with other Burn dependencies in the project.
